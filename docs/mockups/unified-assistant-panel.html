统一助手面板 · 样张（v1）

一个助手，工具域跟随当前活跃区切换。「拆镜头/定妆」不再是按钮，而是说人话就能触发的工具。后端早已单 runtime + 单 session（跨区记忆连续）。光模式 · token-only · 密度优先。

★ 本样张专门回应一个风险：「单面板会让『此刻能用哪些工具』变得不可见」。解法 = 顶栏上下文胶囊 + 顶部工具域条，始终明示你在哪个区、能调哪些工具。

关键设计决定（请逐条看）
  • 上下文胶囊（顶栏右）：始终显示你在哪个区——✍️创作 / 🎨画布 / 🎬时间轴。
  • 工具域条（顶部灰条）：列出此刻可调工具，回应「工具不可见」风险。
  • 拆镜头/定妆 = 工具，不是按钮；说人话触发，删 window 事件桥。
  • 两种定妆文案消歧义：剧本级「立角色卡（无图）」 vs 节点级「基于此图定妆（i2i）」。两条都保留。
  • 模式 + 模型选择器三处一致、常驻，不再时有时无。

当前问题

  • 两个面对用户的助手，控件/模式/模型语义各异（模型选择器只在生成区、还会自我隐藏）。
  • 跨区靠 window CustomEvent + setTimeout(60) 等面板挂载——脆弱、时序耦合、难测。
  • 「拆镜头」「定妆」是制片行话，被摆成一等按钮，用户看不懂；定妆两入口同名，点了不知道得到哪种。
